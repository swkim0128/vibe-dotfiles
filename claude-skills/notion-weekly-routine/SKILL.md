# 주간 일정정리 루틴

매주 정기적으로 진행하는 3단계 주간 정리를 하나의 대화 흐름으로 처리합니다.

## 전체 흐름

```
[1단계] 이번 주 일정 정리   ← 사용자 입력 후 노션 반영
        ↓
[2단계] 이번 주 회고록 작성  ← 1단계 내용 자동 분석 후 작성
        ↓
[3단계] 다음 주 일정 계획   ← 사용자 입력 후 노션 반영
```

각 단계가 끝날 때마다 진행 상황을 표시합니다:
```
✅ 1단계 완료: 이번 주 일정 정리
⏳ 2단계 진행 중: 회고록 작성
⬜ 3단계 대기: 다음 주 계획
```

사용자가 특정 단계만 원하면 해당 단계만 진행합니다.

---

## 준비: 노션 페이지 확인

루틴 시작 전, 이번 주와 다음 주 Diary 페이지를 동시에 찾습니다.

- **데이터베이스**: Diary (`collection://d4c94e28-6040-45f4-a4ae-69b74a6b26b4`)
- **페이지 명명 규칙**: `[week XX] @YYYY/MM/DD → YYYY/MM/DD 일지`
- 이번 주 주차 계산 후 검색 (예: "week 11 2026 일지")
- 다음 주는 이번 주 + 1

두 페이지를 fetch하여 현재 상태(기존 기록 여부, 회고 존재 여부)를 파악한 뒤 루틴을 시작합니다.

페이지가 없으면 아래 설정으로 생성합니다:
```json
{
  "parent": { "data_source_id": "d4c94e28-6040-45f4-a4ae-69b74a6b26b4" },
  "pages": [{
    "properties": {
      "Title": "[week XX] @YYYY/MM/DD → YYYY/MM/DD 일지",
      "Year": "2026",
      "Quarter": "1 분기",
      "Tags": "[\"일상\"]"
    }
  }]
}
```

## 1단계: 이번 주 일정 정리

이번 주에 실제로 했던 일을 노션 페이지에 기록합니다.

### 사용자 입력 요청

```
이번 주(X/XX ~ X/XX)에 있었던 일을 알려주세요.

- 업무(WORK): 어떤 일을 하셨나요?
- 개인(LIFE): 운동, 약속, 개인 활동은요?
- 특정 요일에 있었던 일이 있다면 함께 알려주세요.
```

### 노션 반영 규칙

**페이지 구조**: `notion-weekly-schedule` 스킬의 슬롯 매핑 규칙을 따릅니다.

| 슬롯 | 시간 | 용도 |
|------|------|------|
| LIFE1 | 06-09 | 이른 아침 개인 활동 |
| WORK1 | 09-12 | 오전 업무 |
| WORK2 | 12-15 | 오후 업무 |
| WORK3 | 15-18 | 오후 업무 |
| LIFE2 | 18-21 | 저녁 개인 활동 |
| LIFE3 | 21-24 | 밤 개인 활동 |

주말(토·일)은 WORK 슬롯 없이 LIFE1~LIFE6만 사용합니다.

**Week List 업데이트**: 요일 슬롯과 함께 Week List의 WORK/LIFE 항목도 함께 업데이트합니다.
- WORK: 협력사 / Billing / Other 분류
- LIFE: Sport / Housework / Side Project / Study / Other 분류

**요일 컬럼 교체 시 주의**: 슬롯을 부분 수정하면 다른 슬롯이 삭제되므로, 해당 요일의 전체 슬롯 목록을 new_str에 포함합니다.

반영 후 변경 내용을 표로 요약하여 사용자에게 보여줍니다.

## 2단계: 회고록 작성

1단계 완료 직후 별도 입력 없이 자동으로 진행합니다.

이미 fetch한 페이지 내용을 분석하여 KPT 회고를 작성하고, 페이지 하단에 추가합니다.

### 회고 작성 규칙

`notion-weekly-retrospective` 스킬의 KPT 가이드라인을 따릅니다.

```markdown
## 📝 주간 회고

### Keep (유지할 것) ✅
- **항목 제목**: 구체적인 설명

### Problem (개선할 것) ⚠️
- **항목 제목**: 구체적인 설명

### Try (시도할 것) 🎯
- **항목 제목**: 구체적인 설명

### 한 줄 소감
> 이번 주를 한 마디로 정리하는 문장
```

- Keep: 완료된 항목(`[x]`) 기반, 3~5개
- Problem: 미완료 항목(`[ ]`) 및 아쉬운 점, 2~4개
- Try: Problem에서 도출된 구체적 액션, 2~4개

**삽입 위치**: `</columns>\n<empty-block/>` 다음

이미 회고가 있으면 사용자에게 확인 후 덮어쓰거나 스킵합니다.

회고 작성 후 Keep/Problem/Try 개수를 요약하여 보여줍니다.

---

## 3단계: 다음 주 일정 계획

다음 주 예정 일정을 노션 페이지에 미리 기록합니다.

### 사용자 입력 요청

```
다음 주(X/XX ~ X/XX) 계획을 알려주세요.

- 업무 계획은?
- 개인 일정(운동, 약속 등)은?
- 구글 캘린더도 확인할까요?
```

구글 캘린더 확인을 원하면 `gcal_list_events`로 다음 주 일정을 조회하여 함께 반영합니다.

### 노션 반영 규칙

1단계와 동일한 슬롯 매핑 규칙을 적용합니다.
다음 주 페이지의 현재 상태(fetch로 확인)를 기준으로 업데이트합니다.

---

## 에러 처리

| 상황 | 대응 |
|------|------|
| 이번 주/다음 주 페이지 없음 | 페이지 생성 후 루틴 진행 |
| 이미 회고가 있음 | 사용자에게 확인 (덮어쓰기 / 스킵) |
| 사용자가 단계 스킵 요청 | 해당 단계 건너뛰고 다음으로 |
| 구글 캘린더 일정 없음 | "다음 주 캘린더 일정이 없습니다" 안내 |
