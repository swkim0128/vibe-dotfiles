# PARA Task Review

Obsidian PARA 볼트에서 미완료 작업을 종합적으로 스캔하여 우선순위별 요약을 제공한다.

## When to Use

- "해야할 작업 확인해줘", "할 일 뭐 남았어", "이번 주 할 일"
- "what's pending", "check my tasks", "task review"
- 주간 업무 정리, 다음 주 계획 수립 시

**NOT for:** TASKS.md 단일 파일 관리 (→ task-management 스킬 사용)

### 1. 주간업무 파일 (Master Hub)

`02.Areas/01.주간보고/01.주간해야할 일.md`

이 파일이 **진짜 허브**다. TASKS.md가 아니다.

- `수동 추가 항목` 섹션의 `- [ ]` 체크박스가 실제 작업 목록
- `- [x]` 완료 항목은 제외
- Obsidian Tasks 플러그인 쿼리 블록(` ```tasks `)은 동적이므로 raw 텍스트만으로는 결과를 볼 수 없음 — 수동 항목만 추출
- 태그 분류: `#협력사`, `#빌링`, `#내부`, `#보안`, `#Other` 등
- week 번호와 날짜 범위 확인 → 현재 날짜와 비교하여 이월 여부 판단

### 2. 00.Actions/ (인박스)

모든 `.md` 파일을 읽고 액션 아이템 추출:
- 체크박스 (`- [ ]`)
- 명시적 지시사항 (예: 면담 메모의 "~해야 한다", "~필요")
- 빈 파일이나 참고 테이블은 무시

### 3. 01.Projects/ (진행 중 프로젝트)

**재귀적으로** 모든 하위 디렉토리 포함하여 `.md` 파일 스캔:
- 각 파일의 `- [ ]` 미완료 체크박스 추출
- frontmatter의 `status`, `priority`, `plane-issue` 참고
- 하위 작업노트 디렉토리 (`*_작업노트/`) 포함 스캔
- **완료된 프로젝트 파일** (모든 체크박스 완료, status: done)은 제외
- **코드 리뷰 보류 항목** (보안 취약점, 배포 차단 등) 반드시 포함

## Output Template

아래 구조로 출력한다:

```markdown
## 미완료 작업 현황 ({week} 기준)

### 🔴 즉시 처리 필요
| # | 작업 | 이슈 | 비고 |
|---|------|------|------|
(운영 이슈, 보안 취약점, 마감 임박 항목)

### 🟡 진행 중 프로젝트 상세
**프로젝트명 (이슈번호)** — N개 하위 태스크
- 주요 미완료 항목 나열

### ⚠️ 코드 리뷰/보안 보류
(배포 차단 수준 항목 별도 표시)

### 📋 인박스 메모 액션 아이템
(면담/회의 메모에서 추출한 비체크박스 지시사항)

### 📥 백로그
(우선순위 낮음, 일정 미정)

---
**다음 주 우선순위 제안:**
1. (가장 시급)
2. (병렬 진행 가능 항목)
3. (루틴 작업)
```

## Critical Rules

1. **주간업무 파일이 Master** — TASKS.md보다 우선
2. **코드 리뷰 보안 이슈는 절대 누락하지 않는다** — 배포 차단 수준이면 즉시 처리에 포함
3. **현재 날짜 인식** — week 범위가 지났으면 "이월 작업"으로 명시
4. **다음 주 제안 필수** — 단순 나열이 아닌 우선순위 기반 착수 제안
5. **면담/회의 메모의 액션 아이템** — 체크박스가 아니어도 지시사항이면 포함
6. **병렬 실행** — 00.Actions와 01.Projects 스캔은 서브에이전트로 병렬 처리하여 속도 향상

## Common Mistakes

| 실수 | 올바른 방법 |
|------|------------|
| TASKS.md를 주 허브로 사용 | 주간업무 파일이 Master |
| Tasks 플러그인 쿼리 결과를 기대 | raw 파일에서는 수동 항목만 추출 가능 |
| 완료된 프로젝트 파일까지 스캔 | status 확인 후 완료 건 제외 |
| 코드 리뷰 보류 항목 누락 | 보안/배포 차단은 반드시 별도 섹션 |
| 다음 주 제안 없이 나열만 | 우선순위 기반 착수 제안 필수 |
