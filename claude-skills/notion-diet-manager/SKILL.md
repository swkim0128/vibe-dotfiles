# 노션 식단 관리

이번 주 Diet 페이지를 기준으로 식재료 목록을 최신화하고, 먹은 음식을 기록하는 스킬입니다.

## 데이터베이스 정보

| DB | Collection URL | 용도 |
|----|---------------|------|
| Diet | `collection://ed46300e-3f34-4ad5-8319-100810b659b9` | 주간 식단 페이지 |
| Ingredients | `collection://3f8112d6-3518-4d77-b86c-8e481c687d2f` | 식재료 마스터 |
| Foods | `collection://8b14c31d-85b4-411d-a1ed-538222261d09` | 음식 마스터 |

## 전체 흐름

```
1. 이번 주 Diet 페이지 확인
2. 사용자 입력 분류 (추가할 식재료 / 제거할 식재료 / 먹은 음식)
3. 각 항목 처리 (DB 검색 → 없으면 생성 → 관계 업데이트)
4. 변경 결과 요약
```

---

## 1단계: 이번 주 Diet 페이지 확인

Diet DB에서 이번 주 페이지를 검색합니다.
- **명명 규칙**: `[week XX] @YYYY/MM/DD → YYYY/MM/DD 식단`
- 주차와 날짜를 계산해 검색어를 만들고 `notion-search`로 조회합니다.

**페이지가 없을 때**: 식단 페이지는 매주 일요일에 n8n으로 자동 생성되므로, 없을 경우 사용자에게 안내합니다.
> "이번 주 식단 페이지가 아직 생성되지 않았습니다. n8n 자동화가 실행되면 다시 시도해 주세요."

페이지를 찾으면 `notion-fetch`로 현재 Ingredients / Food 관계 목록을 가져옵니다.

---

## 2단계: 사용자 입력 분류

사용자 입력을 세 가지로 분류합니다.

| 유형 | 설명 | 예시 |
|------|------|------|
| **식재료 추가** | 새로 구매한 식재료 | "계란 샀어", "당근이랑 양파 장봤어" |
| **식재료 제거** | 다 소진한 식재료 | "두부 다 먹었어", "고추장 다 썼어" |
| **음식 기록** | 오늘 저녁에 먹은 음식 | "오늘 김치찌개 먹었어", "저녁에 라면" |

입력이 불명확하면 짧게 확인합니다. 여러 유형이 섞여 있으면 모두 처리합니다.

---

## 3단계: 식재료 추가

새로 구매한 식재료를 Diet 페이지 Ingredients 관계에 연결합니다.

### 검색
`notion-search`로 Ingredients DB(`collection://3f8112d6-3518-4d77-b86c-8e481c687d2f`)에서 이름으로 검색합니다.

### 없으면 생성
Ingredients DB에 없으면 `notion-create-pages`로 생성합니다.
- **Name**: 식재료 이름
- **Tag**: 아래 목록에서 가장 적합한 태그 자동 선택

| 태그 | 해당 품목 예시 |
|------|--------------|
| 야채 | 당근, 양파, 마늘, 파, 배추 등 |
| 정육/계란 | 닭고기, 돼지고기, 소고기, 계란 등 |
| 해산물 | 새우, 오징어, 조개, 생선 등 |
| 국/반찬/메인요리 | 김치, 된장찌개 등 완성된 음식 |
| 면 | 라면, 당면, 소면 등 |
| 소스류 | 간장, 된장, 고추장, 케첩 등 |
| 오일 | 참기름, 올리브오일, 식용유 등 |
| 조미료 | 소금, 설탕, 후추, 다시다 등 |
| 과일 | 사과, 바나나, 딸기 등 |
| 우유/유제품 | 우유, 요거트, 치즈 등 |
| 생수/음료/차 | 생수, 주스, 커피 등 |

### Diet 페이지 업데이트
현재 Ingredients 관계 배열에 새 페이지 URL을 추가해 `notion-update-page`로 반영합니다.
기존 관계 목록을 유지한 채 새 항목만 append하는 것이 핵심입니다.

---

## 4단계: 식재료 제거

소진된 식재료를 Diet 페이지 Ingredients 관계에서 제거합니다.

1. 현재 Diet 페이지의 Ingredients 관계 목록(URL 배열)을 가져옵니다.
2. 제거할 식재료 이름과 일치하는 항목의 URL을 찾습니다.
   - 각 URL을 `notion-fetch`하거나, 이름으로 Ingredients DB에서 검색해 URL을 확인합니다.
3. 해당 URL을 배열에서 제거한 뒤 `notion-update-page`로 Ingredients 관계를 업데이트합니다.

이 때 Ingredients DB의 항목 자체는 삭제하지 않습니다. Diet 페이지와의 연결만 끊습니다.

---

## 5단계: 음식 기록

오늘 **저녁**에 먹은 음식을 Diet 페이지 Food 관계에 연결합니다.
아침/점심은 기록하지 않습니다. "먹었어"라고만 해도 저녁 기준으로 처리합니다.

### 검색
`notion-search`로 Foods DB(`collection://8b14c31d-85b4-411d-a1ed-538222261d09`)에서 이름으로 검색합니다.

### 없으면 생성
Foods DB에 없으면 `notion-create-pages`로 생성합니다.
- **Name**: 음식 이름
- **Tag**: 아래 목록에서 적합한 태그 자동 선택

| 태그 | 기준 |
|------|------|
| 단백질 | 주재료가 고기/생선/계란 위주 |
| 빠른 | 조리가 간단하거나 즉석 음식 |
| 건강한 | 채소 위주, 저칼로리 |
| 무거운 | 기름지거나 칼로리 높음 |
| 주말 | 손이 많이 가는 요리 |
| 양식 | 파스타, 피자, 스테이크 등 |
| 일식 | 초밥, 라멘, 돈카츠 등 |
| 특별한 | 외식, 특별한 날의 음식 |

### Diet 페이지 업데이트
현재 Food 관계 배열에 새 페이지 URL을 추가해 반영합니다.

---

## 결과 요약

모든 처리가 끝나면 아래 형식으로 요약합니다.

```
✅ [week XX] 식단 업데이트 완료

| 구분 | 항목 |
|------|------|
| 식재료 추가 (+N) | 계란, 당근, 양파 |
| 식재료 제거 (-N) | 두부, 고추장 |
| 음식 기록 (+N) | 김치찌개 |
```

새로 생성된 항목은 이름 뒤에 (신규) 표시를 붙여 구분합니다.

---

## 에러 처리

| 상황 | 대응 |
|------|------|
| 이번 주 Diet 페이지 없음 | n8n 자동화 대기 안내 |
| 식재료/음식 이름 모호 | 사용자에게 짧게 확인 |
| 제거 요청 식재료가 목록에 없음 | "현재 목록에 없습니다" 안내 |
| 동일 이름 항목 다수 | 가장 최근 생성 항목 사용 |
