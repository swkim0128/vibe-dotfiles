방금 작성한 코드(또는 지정한 함수/클래스)에 대한 테스트 코드를 생성해줘.

다음 기준을 지킬 것:
1. 해피 패스 1개 + 경계 조건 2~3개 + 예외 경로 1~2개로 구성
2. 이 프로젝트의 기존 테스트 파일(naming, fixture, assertion 스타일)과 **형식/구조가 반드시 동일**할 것
   - 사전에 프로젝트 내 기존 테스트 파일 1~2개를 먼저 읽고 패턴 파악
3. 테스트 케이스명은 "무엇을 검증하는지" 한국어 또는 영어로 명확히
4. mock/stub은 외부 I/O(DB, HTTP, 파일시스템)에만 제한적으로 사용
5. 테스트 실행 명령어도 한 줄로 함께 제시

테스트 이름에 "test1", "test_case_a" 같은 무의미한 이름을 쓰지 말 것.
