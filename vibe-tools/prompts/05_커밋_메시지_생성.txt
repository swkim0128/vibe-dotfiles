현재 스테이지된 변경사항을 기반으로 커밋 메시지를 작성해줘.

먼저 실행할 것:
- `git diff --staged` 출력 확인
- 변경된 파일 목록(`git diff --staged --name-only`) 확인

작성 규칙:
1. Conventional Commits 형식 사용 — `type(scope): subject`
   - type: feat / fix / refactor / docs / test / chore / perf / style
   - subject: 72자 이내, 명령문, 마침표 없음
2. 본문(body) — 필요한 경우에만 작성. "무엇을"보다 "왜"를 설명
3. 변경이 여러 성격을 섞고 있으면 → 커밋 분리 제안 (메시지를 억지로 하나로 합치지 말 것)
4. 현재 브랜치명이나 이슈 번호가 파악되면 메시지 말미에 `Refs: <이슈>` 형식으로 포함

이미 배포된 기능의 버그 수정이면 fix, 신규 기능이면 feat, 기존 기능 개선이면 refactor로 명확히 구분.
